## 2026-01-27 项目初始化

### 上午：项目架构设计
**讨论内容**：
- 用户要求创建AI课程学习路径项目
- 需要支持在线代码执行和编辑
- 要求友好的用户界面

**技术决策**：
1. **前端选择**：原生HTML + CSS + JavaScript
   - 原因：避免复杂依赖，易于部署
   - CodeMirror编辑器提供专业的代码编辑体验
   - 模态框实现课程内容的交互式展示

2. **后端选择**：Python HTTPServer
   - 原因：简单、轻量、易于维护
   - subprocess执行确保安全性
   - 支持CORS，便于前端调用

3. **代码执行方式**：临时文件 + subprocess
   - 原因：避免直接eval的安全风险
   - 临时文件确保隔离性
   - 超时机制防止无限循环

### 下午：基础功能实现
**完成工作**：
- 创建HTML页面结构和CSS样式
- 实现模态框功能和课程导航
- 集成CodeMirror代码编辑器
- 实现代码执行API和结果显示

**遇到问题**：
- 代码块在模态框中不显示
- 点击后才显示，但字符有重叠

### 晚上：问题修复
**问题分析**：
- CodeMirror在隐藏元素中初始化时出现问题
- 模态框打开时编辑器没有正确刷新

**解决方案**：
1. 添加viewportMargin配置
2. 在编辑器初始化后调用refresh()
3. 在模态框打开时延迟刷新所有编辑器

## 2026-01-27 用户体验优化

### 用户反馈收集
**反馈1：代码执行提示不友好**
- 用户意见："Connected to: /execute"提示太技术化
- 影响：用户体验不够友好

**解决方案**：
- 移除技术性连接提示
- 添加友好的默认提示："代码执行成功，无输出结果"
- 只在有实际输出时显示结果

**反馈2：代码编辑器太小**
- 用户意见：编辑器窗口太小，不方便编写代码
- 影响：代码编辑体验差

**解决方案**：
- 设置最小高度为400px
- 支持垂直方向调整大小
- 设置最大高度为800px，防止过大

### 服务器稳定性改进
**问题分析**：
- ConnectionAbortedError导致服务器崩溃
- Unicode编码问题影响错误显示
- 缺少OPTIONS请求支持

**解决方案**：
1. 优雅处理客户端中止连接
2. 使用英文错误消息避免编码问题
3. 添加CORS头和OPTIONS请求处理
4. 完善错误处理机制

## 重要决策记录

### 为什么选择原生JavaScript而非框架
**考虑因素**：
- 项目规模相对较小
- 避免构建复杂度
- 减少学习曲线
- 便于部署和维护

**权衡分析**：
- 优点：简单、快速、无依赖
- 缺点：缺少框架的生态支持
- 结论：当前阶段合适，后续可考虑迁移

### 为什么使用subprocess而非eval
**安全考虑**：
- eval存在代码注入风险
- subprocess提供更好的隔离性
- 临时文件确保沙箱环境

**性能考虑**：
- subprocess性能可接受
- 超时机制防止长时间运行
- 资源清理及时
