# AI课程学习路径项目 - 编码规范

## 概述
本文档定义了AI课程学习路径项目的编码规范和最佳实践。所有贡献者必须遵循这些规范，以确保代码质量和可维护性。

---

## 文件大小限制

### 强制规则
- **所有文件都不允许超过2000行代码**
- 超过2000行的文件必须拆分为多个模块或文件
- 此规则适用于所有类型的文件：HTML、CSS、JavaScript、Python、Markdown等

### 拆分原则
1. **功能模块化**：按功能模块拆分文件
2. **逻辑分层**：按逻辑层次拆分文件
3. **职责单一**：每个文件只负责一个主要功能
4. **易于维护**：拆分后的文件应该易于理解和维护

### 拆分示例
**原始文件（超过2000行）**：
```
script.js (2500行)
```

**拆分后**：
```
script.js (主入口，500行)
ui_handlers.js (UI处理，800行)
data_manager.js (数据管理，600行)
api_client.js (API客户端，600行)
```

## 通用编码规范

### 1. 命名规范

#### 1.1 文件命名
- **HTML文件**：使用小写字母和下划线，例如：`preview_all.html`
- **CSS文件**：使用小写字母和下划线，例如：`styles.css`
- **JavaScript文件**：使用小写字母和下划线，例如：`script.js`
- **Python文件**：使用小写字母和下划线，例如：`combined_server.py`
- **Markdown文件**：使用小写字母和下划线，例如：`readme.md`

#### 1.2 变量命名
- **JavaScript变量**：使用camelCase，例如：`userName`, `totalCount`
- **Python变量**：使用snake_case，例如：`user_name`, `total_count`
- **常量**：使用UPPER_SNAKE_CASE，例如：`MAX_SIZE`, `API_URL`
- **类名**：使用PascalCase，例如：`UserManager`, `DataProcessor`

#### 1.3 函数命名
- **JavaScript函数**：使用camelCase，例如：`getUserData()`, `calculateTotal()`
- **Python函数**：使用snake_case，例如：`get_user_data()`, `calculate_total()`
- **事件处理函数**：使用`on`或`handle`前缀，例如：`onClick()`, `handleSubmit()`

### 2. 代码格式

#### 2.1 缩进
- **缩进方式**：使用4个空格（不使用Tab）
- **HTML/CSS**：使用2个空格
- **JavaScript/Python**：使用4个空格

#### 2.2 行长度
- **最大行长度**：120字符
- **建议行长度**：80-100字符
- 超过最大行长度时，合理换行

#### 2.3 空行
- **函数之间**：2个空行
- **类之间**：2个空行
- **逻辑块之间**：1个空行
- **文件末尾**：1个空行

### 3. 注释规范

#### 3.1 注释原则
- **解释为什么**：而不是解释是什么
- **保持简洁**：避免冗长的注释
- **及时更新**：代码修改时同步更新注释
- **避免废话**：不要注释显而易见的代码

#### 3.2 注释格式
- **单行注释**：使用`//`（JavaScript）或`#`（Python）
- **多行注释**：使用`/* */`（JavaScript）或`""" """`（Python）
- **文档注释**：使用JSDoc或docstring格式

#### 3.3 注释示例
```javascript
// 好的注释：解释为什么这样做
// 使用setTimeout避免阻塞UI线程
setTimeout(() => {
    processData();
}, 0);

// 不好的注释：解释显而易见的事情
// 设置变量x为10
let x = 10;
```

### 4. 代码组织

#### 4.1 文件结构
```
1. 文件头部注释
2. 导入语句
3. 常量定义
4. 全局变量
5. 类定义
6. 函数定义
7. 事件监听器
8. 初始化代码
```

#### 4.2 函数结构
```
1. 函数注释
2. 函数声明
3. 变量声明
4. 逻辑处理
5. 返回值
```

### 5. 错误处理

#### 5.1 异常捕获
- **JavaScript**：使用try-catch-finally
- **Python**：使用try-except-finally
- **异步操作**：使用Promise.catch或async/await

#### 5.2 错误日志
- **记录错误信息**：包括错误类型、位置、堆栈
- **用户友好提示**：向用户显示友好的错误消息
- **调试信息**：在开发环境输出详细调试信息

### 6. 性能优化

#### 6.1 代码优化
- **避免重复计算**：缓存计算结果
- **减少DOM操作**：批量操作DOM
- **使用事件委托**：减少事件监听器数量
- **懒加载**：延迟加载非关键资源

#### 6.2 资源优化
- **压缩文件**：使用压缩后的CSS和JavaScript
- **图片优化**：使用适当格式和尺寸
- **缓存策略**：合理使用浏览器缓存

### 7. 安全性

#### 7.1 输入验证
- **验证用户输入**：防止XSS和SQL注入
- **转义输出**：避免XSS攻击
- **参数化查询**：防止SQL注入

#### 7.2 敏感信息
- **不提交密钥**：不要将API密钥提交到版本控制
- **使用环境变量**：敏感信息存储在环境变量中
- **加密存储**：敏感数据加密存储

## 特定语言规范

### JavaScript规范

#### 1. 变量声明
- **优先使用const**：不会重新赋值的变量
- **其次使用let**：需要重新赋值的变量
- **避免使用var**：避免使用var声明变量

#### 2. 函数定义
- **箭头函数**：优先使用箭头函数
- **函数声明**：在需要提升时使用函数声明
- **默认参数**：使用默认参数简化代码

#### 3. 异步处理
- **Promise**：使用Promise处理异步操作
- **async/await**：优先使用async/await语法
- **错误处理**：使用try-catch处理异步错误

### Python规范

#### 1. 类型提示
- **函数参数**：使用类型提示
- **返回值**：使用返回类型提示
- **复杂类型**：使用typing模块

#### 2. 导入顺序
```
1. 标准库导入
2. 第三方库导入
3. 本地模块导入
4. 每组之间空一行
```

#### 3. 类定义
- **继承**：合理使用继承
- **属性**：使用@property装饰器
- **方法**：使用self作为第一个参数

### HTML规范

#### 1. 文档结构
- **使用语义化标签**：header, nav, main, footer等
- **正确的嵌套**：确保标签正确嵌套
- **闭合标签**：所有标签正确闭合

#### 2. 可访问性
- **alt属性**：为图片添加alt属性
- **标签关联**：label与input正确关联
- **ARIA属性**：适当使用ARIA属性

### CSS规范

#### 1. 选择器
- **避免过度嵌套**：最多3层嵌套
- **使用类选择器**：优先使用类选择器
- **避免ID选择器**：避免使用ID选择器

#### 2. 样式组织
- **按功能分组**：相关样式放在一起
- **使用注释**：分隔不同的样式块
- **响应式设计**：使用媒体查询
