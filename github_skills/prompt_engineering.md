# 系统提示词工程

This skill provides system prompt engineering and optimization techniques for AI models.

## When to Invoke

- User asks for "prompt optimization"
- User requests "system prompt design"
- User wants to "improve prompt quality"
- User mentions "prompt engineering"
- User needs "better AI responses"
- User asks for "system instructions"

## Description

This skill helps with:
- System prompt design and optimization
- Prompt engineering techniques
- Context management
- Output format control
- Response quality improvement

### 核心概念

系统提示词是给AI模型的初始指令，定义了AI的角色、能力和行为边界。

### 提示词工程技巧

通过精心设计提示词，可以显著提高AI的输出质量和一致性。

## Best Practices

### 1. 系统提示词设计原则

#### 清晰性
- 明确定义AI的角色和职责
- 清晰说明AI的能力范围
- 设定明确的行为边界
- 避免模糊不清的指令

#### 具体性
- 提供具体的行为指导
- 避免抽象的描述
- 使用具体的例子说明
- 定义明确的输出格式

#### 一致性
- 保持指令风格的一致性
- 使用统一的术语
- 维持角色的一致性
- 避免相互矛盾的指令

#### 可测试性
- 设计可验证的指令
- 提供评估标准
- 包含质量控制检查点
- 支持迭代改进

### 2. 提示词工程技巧

#### 上下文管理
- 提供相关的背景信息
- 维护对话历史
- 使用变量引用上下文
- 适时更新上下文

#### 输出控制
- 指定输出格式（JSON、Markdown、代码）
- 设置输出长度限制
- 定义结构化输出要求
- 控制详细程度

#### 迭代优化
- 从简单版本开始
- 基于反馈持续改进
- A/B测试不同版本
- 记录优化过程

## Troubleshooting

### 常见问题

#### 1. AI不遵循指令
**症状**：AI的输出不符合系统提示词的要求
**解决方案**：
- 重新审视系统提示词的清晰度和具体性
- 添加更多示例和约束条件
- 使用更明确的指令动词

#### 2. 输出格式不一致
**症状**：AI的输出格式不统一或不完整
**解决方案**：
- 在系统提示词中明确定义输出格式
- 提供输出模板或示例
- 添加格式验证要求

#### 3. AI能力超出范围
**症状**：AI尝试执行超出其能力范围的任务
**解决方案**：
- 在系统提示词中明确设定能力边界
- 提供拒绝或转介的指导
- 添加相关技能的引用

#### 4. 上下文丢失
**症状**：AI在多轮对话中丢失或混淆上下文
**解决方案**：
- 在系统提示词中强调上下文管理
- 提供上下文总结和引用机制
- 使用明确的对话结构
