# GitHub Skills 测试用例

这个文件包含了测试我们创建的GitHub skill的测试用例。

## 测试目标

验证我们创建的GitHub skill文件是否能够：
1. 被Trae AI正确理解和执行
2. 生成高质量的代码和文档
3. 遵循最佳实践
4. 提供实用的解决方案

## 测试用例

### 测试1：AI Agent Skill

**Skill文件**：`github_skills/ai_agent.md`

**测试场景**：创建一个简单的AI Agent

**测试步骤**：
1. 在Trae IDE中打开`github_skills/ai_agent.md`
2. 在Trae AI聊天中输入以下提示词：
   ```
   请帮我创建一个简单的AI Agent，用于代码生成。Agent应该有：
   1. 一个名称和描述
   2. 基本的处理方法
   3. 简单的任务执行能力
   
   请使用Python编写代码，并遵循PEP 8规范。
   ```

**期望结果**：
- ✅ AI理解AI Agent的概念
- ✅ 生成符合要求的Python代码
- ✅ 代码包含适当的注释和文档字符串
- ✅ 代码遵循PEP 8规范

**验证点**：
- [ ] Agent类有清晰的`__init__`方法
- [ ] Agent类有`process`方法
- [ ] Agent类有`capabilities`属性
- [ ] 生成的代码可以运行
- [ ] 代码有适当的错误处理

### 测试2：系统提示词工程Skill

**Skill文件**：`github_skills/prompt_engineering.md`

**测试场景**：优化系统提示词

**测试步骤**：

   请帮我：
   1. 分析当前系统提示词的优缺点
   2. 提供改进建议
   3. 创建一个优化后的系统提示词版本
   4. 解释每个改进的理由
   ```

**期望结果**：
- ✅ AI分析当前系统提示词
- ✅ 提供具体的改进建议
- ✅ 创建优化后的系统提示词
- ✅ 解释每个改进的理由

**验证点**：
- [ ] AI分析了当前系统提示词
- [ ] 提供了改进建议
- [ ] 创建了优化后的系统提示词
- [ ] 解释了改进理由

### 测试3：Skill模板Skill

**Skill文件**：`github_skills/skill_template.md`

**测试场景**：使用模板创建新skill

**测试步骤**：
1. 在Trae IDE中打开`github_skills/skill_template.md`
2. 在Trae AI聊天中输入以下提示词：
   ```
   请帮我使用skill_template.md作为模板，创建一个新的skill文件。
   
   新skill的要求：
   - 名称：code-reviewer（代码审查助手）
   - 描述：帮助进行代码审查，提供改进建议
   - 触发条件：当用户请求代码审查或改进建议时
   - 功能：分析代码质量、提供重构建议、检查最佳实践
   ```

**期望结果**：
- ✅ AI理解skill模板的结构
- ✅ 创建符合模板格式的新skill文件
- ✅ 新skill包含所有必需的部分
- ✅ 新skill有清晰的描述和使用示例

**验证点**：
- [ ] 新skill文件有正确的frontmatter
- [ ] 新skill包含"When to Invoke"部分
- [ ] 新skill包含"Description"部分
- [ ] 新skill包含"Usage Examples"部分
- [ ] 新skill包含"Best Practices"部分

### 测试4：最佳实践Skill

**Skill文件**：`github_skills/best_practices.md`

**测试场景**：应用最佳实践到代码

**测试步骤**：
1. 在Trae IDE中打开`github_skills/best_practices.md`
2. 在Trae AI聊天中输入以下提示词：
   ```
   请帮我根据best_practices.md中的指南，审查以下代码：
   
   ```python
   def calculate_sum(numbers):
       total = 0
       for num in numbers:
           total += num
       return total
   ```
   
   请告诉我：
   1. 这段代码是否符合命名约定
   2. 这段代码是否有适当的文档
   3. 这段代码是否有错误处理
   4. 这段代码如何改进
   5. 提供改进后的代码
   ```

**期望结果**：
- ✅ AI理解最佳实践的概念
- ✅ AI识别代码中的问题
- ✅ AI提供具体的改进建议
- ✅ AI生成改进后的代码
- ✅ AI解释每个改进的理由

**验证点**：
- [ ] AI识别了命名问题
- [ ] AI识别了文档缺失
- [ ] AI识别了错误处理缺失
- [ ] AI提供了改进建议
- [ ] AI生成了改进后的代码
- [ ] 改进后的代码符合最佳实践

## 测试方法

### 方法1：手动测试
1. 在Trae IDE中打开skill文件
2. 在Trae AI聊天中输入测试提示词
3. 观察AI的响应
4. 验证响应是否符合期望
5. 记录测试结果

### 方法2：集成测试
1. 将skill集成到实际项目中
2. 在项目中使用skill
3. 验证功能是否正常工作
4. 记录问题和改进

### 方法3：自动化测试
1. 创建测试脚本
